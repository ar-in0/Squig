◆ GetClientParameters()

ClientParameters * RTMPServerSession::GetClientParameters ( )

Get client parameters received by streamer. This parameters is seted after first GetRTMPMessage() call, before it it does not have any sense. Caller can update parameters by data received from media packets. For example, NGINX RTMP module pushes ActionScript metadata without audio channels count and samplerate

◆ GetRTMPMessage()

RTMPMediaMessage RTMPServerSession::GetRTMPMessage ( )

Get media message. Updates client parameters. Blocks calling thread until audio or video message is received.
